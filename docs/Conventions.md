# Соглашения по навигации, контексту и edge-кейсам

> Источник правды по правилам, которые разбросаны или не описаны во Flow/Spec. Замещает любые расхождения по этим темам.

---

## 1. Сохранение маршрута и контекста при переключении сценария

Правило по умолчанию: **раздел сохраняется, под-контекст сбрасывается осознанно**.

| Раздел        | Что сохраняется                          | Что сбрасывается                                       |
| :------------ | :--------------------------------------- | :----------------------------------------------------- |
| Карта сети    | Сам раздел, год среза, zoom/центр карты  | Выделенный проект (если его нет в новом сценарии)      |
| Проекты       | Раздел, фильтры, сортировка              | Открытая карточка (если проекта нет в новом сценарии)  |
| Сравнение     | Раздел, активная вкладка                  | Корзина сравнения (всегда), grouped bar пересчитывается |
| Ранжирование  | Раздел, выбранные показатели и веса       | Результат расчёта (требуется новый расчёт)              |
| Карточка      | Раздел Проекты, активная вкладка карточки | Сам проект, если его нет в новом сценарии              |

При переходе из системного в пользовательский (или наоборот) поведение идентичное.

## 2. Жизненный цикл session-only сущностей

| Сущность                    | Перезагрузка | Переключение сценария | Выход из приложения |
| :-------------------------- | :----------: | :-------------------: | :-----------------: |
| Корзина сравнения            |     ✗        |          ✗            |          ✗          |
| Активные фильтры списка      |     ✗        |          ✓            |          ✗          |
| Год среза карты              |     ✓        |          ✓            |          ✗          |
| Веса и показатели Ranking    |     ✓        |          ✓            |          ✗          |
| Результат расчёта Ranking    |     ✗        |          ✗            |          ✗          |
| Открытая вкладка карточки    |     ✓        |          ✓            |          ✗          |

✓ — переживает событие, ✗ — сбрасывается.

## 3. Год-слайдер: edge-кейсы

- Если выбранный год **меньше** года запуска проекта — проект отображается на карте серым контуром, в Compare помечен «не запущен», в Ranking исключается из расчёта.
- Если **все** проекты Compare-корзины не запущены в выбранном году — таблица показывает строку «нет данных в этом году», grouped bar скрывается, корзина не очищается.
- Сдвиг года **не удаляет** проекты из корзины Compare и не меняет состав ранжирования.
- Год среза не влияет на список проектов, только на отображаемые на карте/в Compare значения.

## 4. Триггеры инвалидации расчёта (`actual` → `stale_after_changes`)

Любое изменение перечисленных полей переводит расчёт в `stale_after_changes`:

- **Размеры движения и пассажиропоток:** размеры движения, сегментация спроса, тарифы.
- **Грузовые перевозки:** объёмы грузопотоков, перераспределение, тарифы.
- **Инфраструктура:** состав станций/депо, длины участков.
- **Финансовая модель:** CAPEX, OPEX, схема финансирования, год запуска, ставка дисконтирования.
- **Социально-экономические эффекты:** допущения P&L, занятость, эффекты по смежным отраслям.
- **Состав сценария:** добавление/удаление проектов в сценарии.

Изменения, которые **не** инвалидируют расчёт: визуальные настройки, фильтры списка, год среза карты, состав корзины Compare, веса Ranking (инвалидируют только Ranking, не сценарий).

## 5. CopyRequiredDialog

- Любая попытка правки в системном сценарии (включая массовую через Compare/Ranking) открывает CopyRequiredDialog.
- Диалог имеет три действия: «Создать копию и продолжить», «Открыть как read-only», «Отмена».
- При «Создать копию и продолжить» — создаётся пользовательская копия сценария, пользователь переносится в неё с сохранением раздела и под-контекста (по правилам §1), действие правки повторяется автоматически.
- В Ranking и Compare массовые правки до подтверждения копии не применяются ни к одному проекту.

## 6. Возвраты и запрет тупиков

Каждый pop-up, модалка и состояние ошибки обязаны иметь явный путь:

| Источник                                | Действия (минимум)                                |
| :-------------------------------------- | :------------------------------------------------ |
| Pop-up «Информация о сценарии»          | «Закрыть», «Открыть рабочую область сценария»     |
| Модалки Compare (фильтры, настройки)    | «Применить», «Отмена»                             |
| Модалка экспорта                        | «Скачать», «Отмена»                               |
| CopyRequiredDialog                      | «Создать копию и продолжить», «Read-only», «Отмена» |
| Состояние `calculation_error`           | «Повторить расчёт», «Открыть журнал ошибки»       |
| Empty-state любого экрана               | Один основной CTA + ссылка на смежный раздел      |

Запрещено: модалка без кнопок закрытия, ошибка без действия, pop-up, открытый поверх другого pop-up.

## 7. Якоря требований и трассировка

- L0/L1 — высокоуровневые требования, размечаются на уровне разделов.
- L4 — атомарные атрибуты, обязаны иметь идентификатор вида `L4-<area>-<group>.<field>` (например `L4-fin-capex.profile`, `L4-pax-segm.business`).
- Каждый L4 в Spec должен иметь привязку «вкладка → EditGroup → поле».
- Полная карта якорей и привязка к EditGroup — `docs/AtomicAnchors.md`.
- Где якорь ещё не проставлен — это **долг документации**, отслеживается в `backlogsfunction.md`.

# Матрица состояний экранов

> Единый источник правды по состояниям UI. Все экраны и вкладки обязаны поддерживать состояния из этого документа. Терминология статусов расчёта — по `Glossary.md`.

---

## 1. Базовые состояния (применяются ко всем экранам)

| Состояние | Когда                                             | UI-проявление                                                  |
| :-------- | :------------------------------------------------ | :------------------------------------------------------------- |
| loading   | Первичная загрузка данных экрана                  | Skeleton + блокировка действий                                 |
| empty     | Данные загружены, список пуст                     | Empty-state с пояснением и одним основным CTA                  |
| error     | Ошибка загрузки                                   | Error-state с кнопкой «Повторить»                              |
| ready     | Данные загружены, есть содержимое                 | Основной интерфейс                                             |

## 2. Состояния расчёта (только там, где это применимо)

`not_calculated`, `in_calculation`, `actual`, `stale_after_changes`, `calculation_error` — определения в `Glossary.md`.

## 3. Permission-зависимые состояния

| Состояние    | Когда                                                                            | UI-проявление                                          |
| :----------- | :------------------------------------------------------------------------------- | :----------------------------------------------------- |
| read_only    | Системный сценарий или роль без права редактирования                              | Поля disabled, кнопки правки скрыты, баннер read-only  |
| copy_gate    | Попытка правки системного сценария                                                | Открывается CopyRequiredDialog                         |
| denied       | Роль не имеет доступа к экрану                                                    | 403-state со ссылкой на доступные разделы              |

## 4. Покрытие по экранам

### 4.1. Карта сети

| Состояние             | Поддерживается | Примечание                                                   |
| :-------------------- | :------------- | :----------------------------------------------------------- |
| loading               | да             | Skeleton карты                                                |
| empty                 | да             | Сценарий без проектов                                         |
| error                 | да             |                                                                |
| ready                 | да             |                                                                |
| status расчёта        | да             | Бейдж статуса в шапке                                         |
| read_only / copy_gate | да             | При попытке правки в системном сценарии                       |

### 4.2. Список проектов

Все базовые состояния, статус расчёта на уровне строк, read_only/copy_gate.

### 4.3. Сравнение

Все базовые состояния. `empty` — корзина пуста. `partial` — менее 2 проектов: graph скрыт, таблица показывается. Статус расчёта — на уровне ячеек проекта.

### 4.4. Ранжирование

Все базовые. `empty` — нет проектов в сценарии. `not_calculated` — нет результатов TOPSIS. `stale_after_changes` — после правки весов/показателей.

### 4.5. Карточка проекта (по 8 вкладкам)

Каждая из 8 вкладок обязана поддерживать: loading, empty (если данных нет), error, ready, read_only/copy_gate. Статус расчёта отображается на уровне карточки (бейдж в шапке) и не дублируется на каждой вкладке.

| Вкладка                          | Особые состояния                                                  |
| :------------------------------- | :---------------------------------------------------------------- |
| Обзор                            | empty при отсутствии полигона на карте                             |
| Исходные данные                  | empty при отсутствии заполненных параметров                        |
| Пассажирские перевозки           | empty при отсутствии моделирования                                 |
| Грузовые перевозки               | empty при отсутствии грузовой составляющей                         |
| Инфраструктура                   | empty при отсутствии станций/депо                                  |
| Финансовая модель                | `not_calculated` отдельно, графики drawdown/debt только при `actual` |
| Социально-экономические эффекты  | `not_calculated` блокирует таблицы эффектов                        |
| Участники и риски                | empty при отсутствии RACI и рисков                                 |

## 5. Правила переходов

- Любая правка триггерного поля → `actual` → `stale_after_changes`. Список триггеров — в `Conventions.md`.
- Запуск расчёта → текущий статус → `in_calculation` → `actual` или `calculation_error`.
- Системный сценарий → любая попытка правки → `copy_gate`.
- Из `calculation_error` единственный путь — «Повторить расчёт» (приводит к `in_calculation`).

## 6. Запрет на тупики

Все состояния `empty`, `error`, `denied`, `calculation_error`, а также все pop-up экраны обязаны иметь явный путь возврата или повтора. Конкретные правила — в `Conventions.md`.

# Глоссарий статусов и терминов

> Единый источник правды по терминологии. Все остальные документы (Disclaimer, Flow, Spec, Guidelines, backlogsfunction) ссылаются на этот файл. При расхождении формулировок приоритет — у этого файла.

---

## 1. Статусы расчёта проекта/сценария

Машинный идентификатор — единственный канон. UI-лейбл и цвет/иконка — фиксированы.

| Идентификатор          | UI-лейбл (RU)              | Цвет / иконка   | Когда применяется                                                                       |
| :--------------------- | :------------------------- | :-------------- | :-------------------------------------------------------------------------------------- |
| `not_calculated`       | Не рассчитано              | neutral / dot   | Сценарий/проект создан, расчёт ни разу не запускался                                    |
| `in_calculation`       | В расчёте                  | blue / spinner  | Расчёт выполняется в фоне                                                               |
| `actual`               | Актуально                  | green / check   | Расчёт завершён, входные данные не менялись с момента расчёта                           |
| `stale_after_changes`  | Устарело после изменений   | amber / warning | Расчёт завершён, но после него были изменены поля из списка триггеров (см. Conventions) |
| `calculation_error`    | Ошибка расчёта             | red / alert     | Расчёт завершился с ошибкой; требуется повтор                                           |

Запрещены формулировки: «stale», «не пересчитано», «требует обновления», «грязно». Только лейблы из таблицы.

## 2. Сущности

| Термин                | Определение                                                                                          |
| :-------------------- | :--------------------------------------------------------------------------------------------------- |
| Сценарий              | Контейнер набора проектов и параметров. Бывает системный (read-only) или пользовательский            |
| Системный сценарий    | Создан системой (Базовый, Утверждённый). Редактирование запрещено, требуется копия                   |
| Пользовательский      | Создан пользователем или скопирован из системного. Редактирование разрешено в рамках прав            |
| Проект                | Инвестиционный проект (линия/станция/депо/проч.) внутри сценария                                     |
| Корзина сравнения     | Session-only набор проектов, выбранных для Compare в текущем сценарии                                |
| Год среза             | Активный год на год-слайдере карты и сравнения                                                       |
| Год запуска           | Атрибут проекта; до этого года проект считается не запущенным                                        |
| Расчёт                | Полный пересчёт показателей сценария/проекта                                                         |

## 3. Действия

| Термин                  | Определение                                                                                     |
| :---------------------- | :---------------------------------------------------------------------------------------------- |
| Копирование сценария    | Создание пользовательской копии системного сценария через CopyRequiredDialog                    |
| Запуск расчёта          | Перевод сценария/проекта в `in_calculation`                                                     |
| Инвалидация             | Перевод в `stale_after_changes` после правки триггерных полей                                   |
| Быстрый экспорт         | Экспорт без конструктора, форматы Excel/PDF/PNG                                                 |
| Расширенный экспорт     | Экспорт через конструктор отчётов (backlog v2, доступен как stub)                               |

## 4. Запрещённые термины и замены

| Не использовать       | Использовать                       |
| :-------------------- | :--------------------------------- |
| «AHP», «иерархии»     | TOPSIS                             |
| «Sensitivity», «чувствительность ранжирования» | (вне скоупа, не упоминать в UI) |
| «PowerPoint», «.pptx» | (вне скоупа, не упоминать в UI)    |
| «Baseline проект»     | (не используется в Compare)        |
| «Pivot», «сводная»    | «Сравнение» / «Compare»            |

## 5. Имена разделов и вкладок (фиксированы)

Верхний уровень: **Карта сети**, **Проекты**, **Сравнение**, **Ранжирование**.

8 вкладок карточки проекта (точные названия и порядок — соответствуют коду `src/app/screens/ProjectCardScreen.tsx`):
1. Обзор (`overview`)
2. Исходные данные (`source`)
3. Пассажирские перевозки (`passenger`)
4. Грузовые перевозки (`cargo`)
5. Инфраструктура (`infra`)
6. Финансовая модель (`finance`)
7. Социально-экономические эффекты (`social`)
8. Участники и риски (`risks`)

Запрещённые в UI синонимы: «Соц.-экон. эффекты», «Соц.-экон.», «Размеры движения и пассажиропоток» (как имя вкладки), «Участники проекта» (как имя вкладки — это группа внутри «Участники и риски»), «Риски и меры реагирования» (как имя вкладки — это группа внутри «Участники и риски»).

# Модель прав доступа (минимальная версия)

> Источник правды по правам и режимам доступа. Замещает любые упоминания ролей и прав в Disclaimer/Flow/Spec.

---

## 1. Роли

| Роль                  | Кто                                                  | Краткое описание                                         |
| :-------------------- | :--------------------------------------------------- | :------------------------------------------------------- |
| Аналитик              | Базовая роль                                          | Создаёт пользовательские сценарии, правит, считает        |
| Эксперт               | Аналитик с расширенными правами                       | + утверждение проектов внутри пользовательского сценария  |
| Наблюдатель           | Read-only пользователь                                | Только просмотр и быстрый экспорт                         |
| Администратор         | Системная роль                                        | Управление сценариями, в т.ч. системными (вне demo)       |

## 3. Режим «Официальная демонстрация»

Чёткое определение:

- Это не отдельная роль и не отдельный экран. Это **флаг отображения**, который применяется к любому пользовательскому сценарию.
- Когда флаг включён:
  - Сценарий помечен бейджем «Официальная демонстрация».
  - Запрещены любые правки данных (полный read_only независимо от роли, кроме Администратора, который может только снять флаг).
  - Запрещён запуск нового расчёта.
  - Разрешён быстрый экспорт.
  - Разрешено добавление в корзину сравнения и просмотр ранжирования.
- Снятие флага возможно только Администратором и только с явным подтверждением.

## 4. Видимость и баннеры

- Любой read_only-режим (роль, системный сценарий, флаг демонстрации) показывает фиксированный баннер вверху рабочей области с пояснением причины и (если применимо) CTA «Создать копию».
- В пользовательских сценариях без read_only баннер отсутствует.

## 5. Что вне скоупа этой версии

- Тонкая ACL по полям и вкладкам (всё или ничего на уровне сценария).
- Делегирование прав между пользователями.
- Аудит-лог действий (вынесен в backlog).
